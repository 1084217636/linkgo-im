# 16 模拟面试与缺口补档规则

## 面试流程

1. 候选人做 1 分钟介绍。
2. 面试官从架构问到链路。
3. 从链路追到结构体、字段、方法。
4. 注入一个故障，要求说影响、检测、恢复、幂等、边界。
5. 追问技术取舍和未完成项。
6. 给出评分和必须补学清单。

## 评分（100）

- 架构与表达 15。
- 单聊可靠性 20。
- Kafka 群聊 15。
- MySQL/Redis 15。
- 游戏运营/红包/AI 10。
- CI/CD/K8s/监控 10。
- 代码所有权 10。
- 真实性与反思 5。

低于 70：继续背框架；70–84：进入源码定位；85 以上：进入压力面和模拟终面。

## 缺口处理

面试中出现资料未覆盖的问题：

1. 判断是否与项目核心相关。
2. 查当前代码和测试，禁止凭印象补答案。
3. 核心问题新增到对应专题和题库。
4. commit、push，记录新增题。
5. 下一轮重复提问直到闭卷回答正确。

## 回答格式

先给结论；再画链路；说明为什么；最后说边界。超过两分钟仍没结论视为表达不合格。

## 第一轮范围

只考 01–05，不考源码细节：项目介绍、三服务、四组件、登录、单聊、群聊和数据边界。通过后再加入业务、K8s 和结构体。
